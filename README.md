# 1111 School

A generative learning platform. Describe what you want to learn, get a personalized course with lessons, practice activities, and a final assessment.

## What's Built

The full learning loop works end-to-end:

- **Course catalog** — Browse predefined courses or create your own with custom objectives
- **AI generation** — A course describer agent runs first to establish a narrative arc, professional role framing, and pre-set lesson titles, then each objective gets a lesson, practice activity, and scoring rubric — all generated by Claude via PydanticAI agents
- **Generation progress UI** — Real-time stepper showing a "Describing your course..." phase followed by per-objective progress (REST + SSE); lesson titles from the describer appear in the stepper as generation proceeds
- **Portfolio-first learning** — Every activity produces an authentic work artifact (drafts, plans, analyses, checklists). Activities are framed around professional scenarios with employer skill signals and quality checklists
- **Lesson reader** — Markdown-rendered lessons with syntax highlighting, sequential navigation, "learn then apply" format with workplace examples
- **Activity submission** — Write a response, get AI-scored feedback including portfolio readiness assessment, employer relevance notes, and resume bullet seeds
- **Revision loop** — After meeting mastery, learners are encouraged (but never required) to revise for portfolio quality. Mastery always auto-unlocks the next lesson
- **Portfolio page** — View all portfolio artifacts across courses, filter by status (draft / revised / portfolio ready / tool ready), promote artifacts, see resume bullet seeds
- **Capstone assessment** — A performance task synthesizing prior artifacts, replacing traditional quiz-style assessment. Passing generates a capstone portfolio artifact
- **Career profile** — Set career interests, target roles, and portfolio goals in Settings to personalize course generation
- **Lesson progression** — Completing an activity unlocks the next lesson
- **Course management** — List, resume, delete courses
- **Sidebar navigation** — Professional role banner, final portfolio outcome goal, capstone progress tracker, and per-lesson activity sub-links with live status

### What to Review

1. **Basic UX/UI** — Is the flow intuitive? Where do you get stuck or confused?
2. **Content quality** — Are the generated lessons useful? Do they actually teach? Or does it feel like slop?
3. **Learning experience** — Does the arc of lesson → activity → feedback → next lesson feel right?
4. **Activity grading** — Is the AI feedback helpful? Are scores fair? Does the rubric make sense?

### What's Still TODO

- **Asset generation** — No images, diagrams, or visual aids in lessons (text-only)
- **Model support** — Currently hardcoded to Claude (Sonnet). No model selection or provider switching.
- **Productionization** — No rate limiting.

## Prerequisites

You'll need an **[Anthropic API key](https://console.anthropic.com/)** with a few dollars of credits loaded. Generating one course costs ~$0.60–$0.80 (~16 LLM calls: 1 course describer + 3 per objective).

**For Docker (quickest way to try it):** just [Docker](https://www.docker.com/get-started/).

**For development:** Docker + [uv](https://docs.astral.sh/uv/) (`curl -LsSf https://astral.sh/uv/install.sh | sh`) + [Node.js 18+](https://nodejs.org/).

## Running

### Docker (quickest)

```bash
./scripts/configure.sh   # one-time: prompts for your Anthropic API key
docker compose up --build
```

Open **http://localhost:8001**. The container reads your API key from `backend/.env`, builds the frontend into the image, runs migrations on start, and serves everything from one port.

### Development

```bash
./scripts/setup.sh   # one-time: installs deps, runs migrations, prompts for API key
npm run dev
```

This starts both the backend (FastAPI on `:8000`) and frontend (Vite on `:5173`) concurrently with hot reloading.

Open **http://localhost:5173** to start.

### First run

1. Browse the catalog or click **Create Your Own**
2. Enter a topic description and 2-5 learning objectives
3. Hit **Generate** and watch the stepper — first a "Describing your course..." step runs, then lessons generate in parallel; a full course takes ~1-2 minutes
4. Once complete, click **Start Learning** to read the first lesson
5. After reading, complete the practice activity and submit
6. Work through all lessons, then take the final assessment

## Testing

The backend has a suite of 66 tests covering the generation pipeline, state machine, SSE tracker, and agent schemas. No real API key or database required — all external calls are mocked.

```bash
cd backend
uv sync --extra dev          # install dev dependencies (one-time)
PYTHONPATH=src ANTHROPIC_API_KEY=test-key uv run pytest tests/ -v
```

Tests live in `backend/tests/`. Each file has a comment at the top explaining what to update when the corresponding source code changes.

## Deploying to AWS

The `infra/` directory contains Terraform to deploy the app on AWS App Runner with an RDS PostgreSQL database.

### What it creates

- **ECR repository** — hosts the Docker image
- **App Runner service** — runs the container (no VPC attachment, so it has direct internet access for Anthropic API calls)
- **RDS PostgreSQL 16** — publicly accessible database (password-protected)
- **VPC + subnets** — required by RDS, with an internet gateway for public access
- **IAM role** — lets App Runner pull images from ECR

### Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) (v1.5+)
- [AWS CLI v2](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html)

### Setup

1. Authenticate with AWS:
   ```bash
   aws login
   ```

2. If using `aws login` (session-based auth), Terraform can't read login sessions directly. Add a `credential_process` bridge profile to `~/.aws/config`:
   ```ini
   [profile terraform]
   credential_process = aws configure export-credentials --profile default --format process
   region = us-east-1
   ```
   If using access keys (`aws configure`), you can skip this step.

3. Create `infra/terraform.tfvars` with your secrets (this file is gitignored):
   ```hcl
   anthropic_api_key = "sk-ant-..."
   jwt_secret        = "some-random-secret"
   db_password       = "some-db-password"
   aws_profile       = "terraform"   # omit if using access keys
   ```

4. Initialize and apply:
   ```bash
   cd infra
   terraform init
   terraform apply
   ```
   This provisions the ECR repo, App Runner service, and RDS database.

5. Configure GitHub Actions secrets using the Terraform outputs — see [CI/CD in CONTRIBUTING.md](CONTRIBUTING.md#cicd).

### Redeploying

Merging to `main` deploys automatically via GitHub Actions. See [CI/CD in CONTRIBUTING.md](CONTRIBUTING.md#cicd) for setup and details.

**Manual fallback:** Run `./infra/deploy.sh` from the repo root.

### Tearing down

```bash
cd infra
terraform destroy
```

## Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `DATABASE_URL` | No | `postgresql+asyncpg://learn:learn@localhost:5432/learn` | PostgreSQL connection string |
| `ANTHROPIC_API_KEY` | Yes | — | Your Anthropic API key |
| `DEFAULT_MODEL` | No | `anthropic:claude-sonnet-4-6` | LLM model identifier (used for lesson planner, writer, activity creator, reviewers) |
| `FAST_MODEL` | No | `anthropic:claude-haiku-4-5-20251001` | Faster/cheaper model used for the course describer agent |
