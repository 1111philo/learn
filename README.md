# 1111 School

A generative learning platform. Describe what you want to learn, get a personalized course with lessons, practice activities, and a final assessment.

## What's Built

The full learning loop works end-to-end:

- **Course catalog** — Browse predefined courses or create your own with custom objectives
- **AI generation** — Each objective gets a lesson, practice activity, and scoring rubric, generated by Claude via PydanticAI agents
- **Generation progress UI** — Real-time stepper showing generation progress per objective (REST + SSE)
- **Lesson reader** — Markdown-rendered lessons with syntax highlighting, sequential navigation
- **Activity submission** — Write a response, get AI-scored feedback with strengths/improvements/tips, retry if desired
- **Lesson progression** — Completing an activity unlocks the next lesson
- **Final assessment** — After all lessons, a summative assessment covers every objective
- **Course management** — List, resume, delete courses

### What to Review

1. **Basic UX/UI** — Is the flow intuitive? Where do you get stuck or confused?
2. **Content quality** — Are the generated lessons useful? Do they actually teach? Or does it feel like slop?
3. **Learning experience** — Does the arc of lesson → activity → feedback → next lesson feel right?
4. **Activity grading** — Is the AI feedback helpful? Are scores fair? Does the rubric make sense?

### What's Still TODO

- **Learner profiles & customization** — The profile form exists but doesn't meaningfully shape generated content yet
- **Asset generation** — No images, diagrams, or visual aids in lessons (text-only)
- **Model support** — Currently hardcoded to Claude (Sonnet). No model selection or provider switching.
- **Productionization** — No rate limiting.

## Prerequisites

You'll need an **[Anthropic API key](https://console.anthropic.com/)** with a few dollars of credits loaded. Generating one course costs ~$0.60 (~15 LLM calls).

**For Docker (quickest way to try it):** just [Docker](https://www.docker.com/get-started/).

**For development:** Docker + [uv](https://docs.astral.sh/uv/) (`curl -LsSf https://astral.sh/uv/install.sh | sh`) + [Node.js 18+](https://nodejs.org/).

## Running

### Docker (quickest)

```bash
./scripts/configure.sh   # one-time: prompts for your Anthropic API key
docker compose up --build
```

Open **http://localhost:8000**. The container reads your API key from `backend/.env`, builds the frontend into the image, runs migrations on start, and serves everything from one port.

### Development

```bash
./scripts/setup.sh   # one-time: installs deps, runs migrations, prompts for API key
npm run dev
```

This starts both the backend (FastAPI on `:8000`) and frontend (Vite on `:5173`) concurrently with hot reloading.

Open **http://localhost:5173** to start.

### First run

1. Browse the catalog or click **Create Your Own**
2. Enter a topic description and 2-5 learning objectives
3. Hit **Generate** and watch the stepper — objectives generate in parallel, a full course takes ~1-2 minutes
4. Once complete, click **Start Learning** to read the first lesson
5. After reading, complete the practice activity and submit
6. Work through all lessons, then take the final assessment

## Testing

The backend has a suite of tests covering the generation pipeline, state machine, SSE tracker, and agent schemas. No real API key or database required — all external calls are mocked.

```bash
cd backend
uv sync --extra dev          # install dev dependencies (one-time)
PYTHONPATH=src ANTHROPIC_API_KEY=test-key uv run pytest tests/ -v
```

Tests live in `backend/tests/`. Each file has a comment at the top explaining what to update when the corresponding source code changes.

## Deploying to AWS

The `infra/` directory contains Terraform to deploy the app on AWS App Runner with an RDS PostgreSQL database.

### What it creates

- **ECR repository** — hosts the Docker image
- **App Runner service** — runs the container (no VPC attachment, so it has direct internet access for Anthropic API calls)
- **RDS PostgreSQL 16** — publicly accessible database (password-protected)
- **VPC + subnets** — required by RDS, with an internet gateway for public access
- **IAM role** — lets App Runner pull images from ECR

### Prerequisites

- [Terraform](https://developer.hashicorp.com/terraform/install) (v1.5+)
- [AWS CLI v2](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html)

### Setup

1. Authenticate with AWS:
   ```bash
   aws login
   ```

2. If using `aws login` (session-based auth), Terraform can't read login sessions directly. Add a `credential_process` bridge profile to `~/.aws/config`:
   ```ini
   [profile terraform]
   credential_process = aws configure export-credentials --profile default --format process
   region = us-east-1
   ```
   If using access keys (`aws configure`), you can skip this step.

3. Create `infra/terraform.tfvars` with your secrets (this file is gitignored):
   ```hcl
   anthropic_api_key = "sk-ant-..."
   jwt_secret        = "some-random-secret"
   db_password       = "some-db-password"
   aws_profile       = "terraform"   # omit if using access keys
   ```

4. Initialize and apply (this builds and pushes the Docker image automatically):
   ```bash
   cd infra
   terraform init
   terraform apply
   ```

### Redeploying

**Automated (recommended):** Merge to `main` — GitHub Actions runs tests, builds the Docker image, pushes to ECR, and triggers App Runner automatically.

**Manual:** Run `./infra/deploy.sh` from the repo root. Builds a new image, pushes to ECR, and triggers an App Runner deployment.

### Tearing down

```bash
cd infra
terraform destroy
```
